**実装方針**
- actor の circuit breaker は公開 API を増やさず、内部用の小さな now-provider を入れて `Instant::now()` 依存を外します。`CircuitBreaker::new` / `CircuitBreakerShared::new` は既定の実装を維持し、テストだけ manual clock を注入します。
- actor の `std/scheduler/tick/tests.rs` は Tokio interval 実装そのものを検証しているため、実時間 sleep ではなく paused Tokio clock に寄せます。これに伴い [modules/actor/Cargo.toml](/Users/j5ik2o/Sources/j5ik2o.github.com/j5ik2o/fraktor-rs/modules/actor/Cargo.toml) の dev `tokio` に `test-util` を追加するのが最小です。
- `std/system/base/tests.rs` の 20ms 待ちは不要なので削除します。`quickstart_with_applies_config_override` は既に sleep なしなのでそのままです。[base/tests.rs](/Users/j5ik2o/Sources/j5ik2o.github.com/j5ik2o/fraktor-rs/modules/actor/src/std/system/base/tests.rs#L39)
- remote の `quickstart` / `multi_node` は固定 sleep をやめ、状態スナップショットやイベント到達を待つ helper に置き換えます。待ち方の既存パターンは [event_stream.rs](/Users/j5ik2o/Sources/j5ik2o.github.com/j5ik2o/fraktor-rs/modules/actor/tests/event_stream.rs#L108) を踏襲します。
- `endpoint_transport_bridge` は production code を大きく触らず、テスト側を paused Tokio clock と condition wait に寄せます。これは `bridge.rs` の非同期ループを全面的に scheduler-backed delay へ置換するより影響が小さく、今回の目的に対して十分です。[bridge.rs](/Users/j5ik2o/Sources/j5ik2o.github.com/j5ik2o/fraktor-rs/modules/remote/src/std/endpoint_transport_bridge/bridge.rs#L138)
- `tokio_tcp` / `tokio_gossip_transport` は実 transport 契約確認として残し、固定 sleep を retry / poll-until に置換します。fast unit から外す必要がある場合だけ、対象 test module を専用 feature で gate します。
- `ci-check` は `unit-fast`、`integration`、`long-running` を追加し、`all` は full のまま `unit-fast` を先に通す構成にします。sleep 検査はまず grep ベースで `thread::sleep` / `tokio::time::sleep` を unit パスで禁止し、integration allowlist を明示します。
