# Pekko ギャップ分析レビュー指摘修正

レビュー指摘に基づき、ギャップ分析レポートを修正してください。

## やらないこと (Do Not)
- ビルドコマンド（`cargo check` / `cargo build` / `cargo test`）を実行しないこと

## レポートファイル一覧

タスクディレクトリ内の以下のレポートを対象とする：

| ファイル | 内容 |
|----------|------|
| `00-api-inventory.md` | Layer 1: API インベントリ |
| `01-interface-comparison.md` | Layer 2: インターフェース比較 |
| `02-design-comparison.md` | Layer 3: 設計比較 |
| `03-pekko-gap-analysis.md` | 統合レポート |
| `04-pekko-gap-review.md` | レビュー結果（指摘元） |

## 手順

1. `04-pekko-gap-review.md` からレビュー指摘を読み込む
2. 指摘された Layer のレポートを修正する：
   - Layer 1 の指摘 → `00-api-inventory.md` を修正
   - Layer 2 の指摘 → `01-interface-comparison.md` を修正
   - Layer 3 の指摘 → `02-design-comparison.md` を修正
3. **修正内容をソースコードで検証する**: 判定を変更した場合は、必ず実際のソースファイルを Read で確認すること
4. 修正した Layer の下流レポートも更新する（例: Layer 1 修正 → Layer 2, 3, 統合も更新）
5. `03-pekko-gap-analysis.md`（統合レポート）のサマリー・数値を再集計する

## 修正原則

- 指摘に直接関係する修正だけを行う
- 判定基準（同名実装 / 別名実装 / 部分実装 / 未実装 / n/a）を再確認すること
- 難易度の根拠が不十分と指摘された場合は、具体的なファイル/シンボルを補足すること
- カバレッジ数値を修正した場合はサマリーテーブルも更新すること
- Layer 間の整合性を確認してから修正完了と判定すること

## 判定基準

- 全指摘に対応完了: `修正完了`
- 対応不能な指摘がある: `一部未対応`
- 判断不能: `判断不能`
