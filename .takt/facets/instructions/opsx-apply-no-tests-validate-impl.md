実装が設計・タスクに合致しているかを検証せよ。

**注意:** OpenSpec の変更ディレクトリ（`openspec/changes/{change}/`）に設計・タスクが揃っていない場合は ABORT する。

**やること:**
1. レポートディレクトリの 00-plan.md から対象変更を特定する
2. `openspec/changes/{change}/` の成果物を読み込む:
   - `proposal.md` - 提案
   - `design.md` - 設計
   - `tasks.md` - タスクリスト
3. タスク完了状況を確認する:
   - `tasks.md` のチェックボックス `[x]` を数える
   - 未完了タスクを特定する
4. 設計整合性を検証する:
   - 設計のディレクトリ構造・Cargo.toml 構成と実装が一致するか確認する
   - example の内容が設計の概要に沿っているか確認する
5. ビルド確認:
   - ビルドコマンドを実行し、全 example がビルドできることを確認する
6. 既存ビルドへの影響:
   - 既存モジュールのビルドが壊れていないか確認する

**注意: テストについて**
- このピースはテスト先行が不要なタスク（examples作成等）向け
- テスト実行ではなくビルド確認を行う
- examples は実行可能であることを `cargo run` で確認する

**必須出力（見出しを含める）**
## タスク完了状況
- 完了: {N} / 全体: {M}
- 未完了タスク: {一覧}
## 設計整合性
- {設計との一致/不一致の詳細}
## ビルド結果
- {実行コマンドと結果}
## 判定
- {検証合格 / 検証不合格}
