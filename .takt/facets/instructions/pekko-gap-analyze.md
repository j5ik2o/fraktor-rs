# Pekko ギャップ分析指示

`{task}` で指定されたモジュールについて、Apache Pekko 参照実装との差分を分析してください。

## 手順

1. 引数からモジュール名を特定し、以下のパスを導出する
   - **fraktor-rs側**: `modules/{name}/src/`
   - **Pekko側**: `references/pekko/{name}/src/`（ディレクトリ名が異なる場合は `references/pekko/` 配下を確認）
2. Pekko側の公開型・トレイト・メソッドを抽出し、`private` / `private[...]` を除外する
3. fraktor-rs側の公開型・トレイト・メソッドを抽出する
4. 両者を以下のカテゴリで分類する
   - 型・トレイト / オペレーター / マテリアライゼーション / グラフDSL / ライフサイクル / エラー処理 / その他
5. ギャップを特定する（判定基準: 実装済み / 別名で実装済み / 部分実装 / 未実装）
6. 各ギャップに難易度を付与する（trivial / easy / medium / hard / n/a）
7. 実装優先度を Phase 1〜4 + 対象外で提案する

## 判定基準

- 名前だけでなくシグネチャ・セマンティクスを総合的に確認すること
- `n/a` 判定は保守的に（JVM固有、Akka互換層、deprecated機能のみ）
- Rust/no_std 固有の制約（`cfg_std_forbid` lint等）を考慮すること
- Pekkoの全機能を移植することが目的ではない（YAGNI原則）

## 判定

- 分析完了、ギャップ一覧と優先度を出力: `分析完了`
- 対象ディレクトリが存在しない: `対象不明`
- 判断不能: `判断不能`
