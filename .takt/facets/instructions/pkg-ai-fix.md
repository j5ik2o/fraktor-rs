# AIレビュー修正（パッケージ構造リファクタリング）

Report Directory 内の `ai-review.md` を一次情報として、AIレビュー指摘を修正せよ。

## やること

1. レポートディレクトリから `ai-review.md` を読み込む
2. レビューレポートが存在しない場合は ABORT する
3. `new` または `persists` の指摘を一覧化する
4. 各指摘に対して以下のいずれかで対処する
   - **修正実施**: 該当ファイルを開き、指摘された問題を修正する
   - **修正不要**: 指摘が誤りである技術的根拠を示す
5. パッケージ構造リファクタリングでは、修正後に **正準パスへ収束していること** を確認する。コンパイル維持のために `#[path = "..."]`、互換 `pub use`、wrapper mod、二重維持を追加してはならない
6. 修正後、`plan.md` またはタスクが指定した `./scripts/ci-check.sh` の範囲を **順番に** 実行し、結果を `coder-decisions.md` の「実行結果」に記録する
7. `coder-decisions.md` の「構造健全性セルフチェック」を更新する

## 修正の原則

- レビュー指摘に直接対応する修正のみ行う（スコープを広げない）
- 既存のコーディング規約・テストパターンを維持する
- no-shim で修正できない根本問題がある場合は、互換レイヤでごまかさず報告する

## やらないこと（重要）

- `cargo check` / `cargo build` / `cargo test` / `cargo clippy` を直接実行しないこと。必ず `./scripts/ci-check.sh` 経由で実行すること
- `ci-check.sh` を複数同時に実行しないこと（cargo ロック競合が発生する）。前のコマンドの完了を必ず待ってから次を実行すること
- `#[path = "..."]`、`include!`、旧パス互換 `pub use` / `pub type` / wrapper mod を追加しないこと

## 修正完了条件（必須）

- ブロッキング指摘をすべて解消したこと
- `coder-decisions.md` の構造健全性セルフチェックが更新されていること
- `plan.md` またはタスクが指定した `ci-check` 範囲がすべてパスしていること

## 判定基準

- 全指摘を修正した → 「AI問題の修正完了」
- 指摘が誤りであると根拠を示せる → 「修正不要（指摘対象ファイル/仕様の確認済み）」
- 修正すべきか判断できない → 「判断できない、情報不足」
