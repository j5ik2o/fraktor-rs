# use-placement-lint 仕様

## 概要
- 目的: `use` と `mod` の宣言はファイルまたはモジュール冒頭にまとまっていないと視認性が低下するため、散在している場合にエラーを出す。
- 対象: Rust ソースファイル全体。マクロ展開によるコードは除外する。

## ルール
1. ファイルまたは inline module の先頭から始まる連続した `use` / `mod` 宣言ブロック以降で、再度 `use` または (テストモジュール以外の) `mod foo;` 形式の宣言が登場したらエラー。
   - `mod foo { ... }` のように定義をインラインで記述する宣言自体は対象外。
   - インラインモジュール内部の `use` / `mod` は、そのモジュール冒頭の宣言ブロックとして判定する。
2. ドキュメンテーションコメントやアトリビュートを伴う宣言でも、実体が `use` / `mod foo;` なら同様に判定する。
3. 関数、メソッド、const block などのブロック内に `use` が登場したらエラー。
   - 必要な import は、現在のファイルまたは enclosing module の冒頭 `use` ブロックへ移動する。

## 推奨修正例
- 問題行に対して「ファイルまたはモジュール冒頭に移動してください」とメッセージを出す。
- AI 用メモ: `use` や `mod` をコメントや属性ごと冒頭に再配置し、その他の宣言より前にまとめる。
- ブロック内 `use` については、対象 `use` と必要な参照パス調整だけを変更する。
