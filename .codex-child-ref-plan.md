# ChildRef 変更計画

1. `ChildRef` の public write API を棚卸しし、`tell` / `try_tell` / `ask` / `ask_with_timeout` を `&mut self`、query を `&self` として境界を確定する。
2. `TypedChildRef` と `TypedActorContext` / `ActorContext` の `ChildRef` 経由 API を `&mut` 契約へ追従させる。
3. `ChildRef` を返す call site と tests を洗い、clone 後に `mut` で受ける形へ最小修正する。
4. `ChildRef` 変更の影響で崩れる `ActorRef` / `TypedActorRef` の補助 API があれば局所修正する。
5. 対象範囲の `cargo check` / 関連テストで確認し、その後 `./scripts/ci-check.sh ai all` で全体確認する。
