## ユーザー入力

```
$ARGUMENTS
```

## 目的
- 仕様タスクを順番に実装し、テストとタスク更新で進捗を確定させる。

## 手順
1. `.kiro/specs/<feature>/tasks.md` を確認し、未完了タスクを洗い出す。
2. タスクを順番に実装し、必要なテストを追加・更新する。
3. 実装完了ごとにタスクのチェックを更新する。
4. 重要な変更は関連ドキュメントにも反映する。
5. 実装後に関連テストを実行し、結果を報告する。
