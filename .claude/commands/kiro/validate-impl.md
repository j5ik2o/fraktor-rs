## ユーザー入力

```
$ARGUMENTS
```

## 目的
- 実装がタスクと要件に沿っているかを検証し、残作業を明確化する。

## 手順
1. `.kiro/specs/<feature>/tasks.md` と該当コードを照合する。
2. 主要な差分・未完了項目・テスト不足を列挙する。
3. 実装が完了している場合は、最終確認の結果を簡潔にまとめる。
